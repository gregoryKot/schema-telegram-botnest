# Правила идеальных тестов (свод практики этого проекта)

## Что тестировать
1. **Тестируй поведение, не реализацию.** Имя теста — утверждение о поведении
   («повторное использование refresh → отзыв всей семьи»), не «тест метода X».
   Ассерти результат/вызовы наружу, не внутренние шаги.
2. **Негативные кейсы обязательны для security-кода**: подделанная подпись,
   протухший токен, чужой userId, отсутствие креды, malformed input. Тест
   «happy path проходит» без «подделка отклоняется» — полтеста.
3. **Integration-тест на wiring.** Юнит гварда не доказывает, что гвард навешан
   на роут. Для каждого контроллера — supertest-тест: 401 без креды, userId из
   токена (не из тела), анти-эскалация полей (белые списки).
4. **Read-after-write** (правило из CLAUDE.md): фича пишет в одном месте, читает
   в другом → тест «сохранил → нашёл», а не только «сохранилось».
5. **Регресс на каждый прод-баг** — воспроизводящий сценарий + комментарий-ссылка
   на инцидент сверху файла. Тест обязан падать ДО фикса.

## Как писать
6. **Самообновляющиеся реестры.** Тест полноты (каскад удаления, merge-таблицы)
   импортирует список из кода (`USER_DATA_TABLES` экспортирован именно для
   этого), а не дублирует его — новая таблица попадает под тест автоматически.
7. **SQL ассертим строкой** там, где семантика оператора критична:
   `IS DISTINCT FROM` vs `<>` на nullable-колонках (`NULL <> x → NULL` = молчаливая
   потеря строк при merge). Паттерн — captureRawSql в `merge.service.spec.ts`.
8. **Детерминизм времени.** Аналитика/расписания — `jest.useFakeTimers()` +
   `setSystemTime` (вернуть в afterEach). TOTP — НЕ fake timers (вешает QRCode),
   а пиновка `authenticator.options.epoch`. Тихие часы — мок `isQuietHours`.
9. **Прямая инстанциация сервисов** с замоканной Prisma (`makePrisma()`),
   без Nest-модуля. TestingModule — только для контроллеров (supertest).
10. **bigint-литералы** (`5n`) в вызовах сервисов — сигнатуры на bigint, number
    в тесте маскирует ошибки типов.
11. **ESM-ловушка**: всё, что транзитивно тянет `jose`, мокает его
    (`jest.mock('jose', …)`) — jest не парсит ESM из node_modules.
12. **Ошибки внешнего мира**: различай permanent (403 blocked → markSent +
    blocked) и transient (наш баг → НЕ markSent, ретрай). Не-фатальность
    уведомлений: падение fetch-алерта не роняет операцию.
13. **Шифрование**: в test-env encrypt = identity (нет ключа). Реальный AES —
    только `crypto.spec.ts` (перезагрузка модуля с env через resetModules).
    Сервисные тесты проверяют «поле прошло через encrypt/decrypt», не сам шифр.
14. **Фронт**: поведение глазами пользователя (`getByRole`/`getByText`,
    `userEvent`), не стили и не пиксели. Хуки — `renderHook` (+ `MemoryRouter`
    для роутерных). fetch-мок — **async-обёртка** (api использует и await, и
    `.then()`).
15. **Гигиена**: один it — одна мысль; без if/циклов вокруг expect (нужны
    кейсы — `it.each`); изоляция (cleanup/localStorage.clear в setup);
    `.skip` без тикета запрещён; `npm test` зелёный перед каждым коммитом.

## Честность покрытия
16. Проценты — не цель, цель — закрытые риски. «100%» для чистых функций —
    буквально; для оркестраторов — все ветви решений; для тонких CRUD-обёрток
    и однострочных делегаций допустим smoke. Недостижимые защитные строки
    помечаем в плане, а не выкручиваем тестами-ради-процентов.
17. **Храповик**: пороги покрытия в конфигах только растут.
