# Требования к статьям сайта

Документ для автора (и для будущих сессий Claude) — чеклист качества для статей
раздела `/articles`. Написан после клинического ревью первых 7 статей практикующим
специалистом по схема-терапии. Статьи редактируются через `/admin` → Статьи.

## Клиническая точность

- **Не выдумывать исследования и цифры.** Если ссылаешься на РКИ/мета-анализ —
  только реальные, проверяемые (пример уже в статьях: Giesen-Bloo et al., 2006 —
  РКИ схема-терапии vs психодинамическая терапия при ПРЛ; Masley et al., 2012 —
  систематический обзор доказательной базы). Если не уверен, что исследование
  существует именно с такими авторами/годом — переформулируй без точной ссылки
  («исследования показывают») вместо угадывания цитаты.
- **Разделять типы травмы, тревоги, депрессии** по тяжести и не смешивать
  «дискомфорт» с клинической картиной. Для тревоги/депрессии/травмы обязательно
  различать: острое/лёгкое состояние (может решиться коротким методом или не
  требует терапии вовсе) vs хроническое/укоренённое (где раскрывается
  схема-терапия). Без этого разделения статья превращается в рекламу метода
  «от всего», что не соответствует действительности и вредит доверию.
- **Не переобещать.** Схема-терапия — не панацея. В каждой статье, которая
  утверждает эффективность метода, должен быть раздел «когда сначала нужно
  другое» или эквивалент — конкретные случаи, когда метод не первая линия
  (острый кризис, изолированная свежая травма, точечный симптом, нет ресурса
  на долгую работу).
- **Доказательная база честно, без прикрас.** У схема-терапии сильнее всего
  доказательства при ПРЛ и хронических паттернах; при остром ПТСР первой линией
  остаются EMDR/травмофокусированная КПТ; при лёгкой/острой депрессии — КПТ
  или медикаменты. Формулировки уровня «метод лечит всё» — красный флаг,
  переписывать.

## Безопасность читателя (обязательно)

- **Любое упоминание суицидальных мыслей, острого кризиса, риска для жизни —
  обязано сопровождаться конкретным путём получения немедленной помощи**:
  горячая линия, направление к психиатру/скорой. Сейчас в статьях используется
  формулировка с номером 8-800-2000-122 (горячая линия экстренной
  психологической помощи, бесплатно, круглосуточно) + скорая 103.
  **Перед публикацией новой статьи с такой темой — проверь, что номер актуален**
  (горячие линии иногда меняются); не копировать вслепую без сверки.
- Никогда не давать читателю самому ставить себе диагноз с уверенностью —
  только «если узнаёте себя» / «похоже на» + рекомендация обсудить со
  специалистом. Формулировки уровня «если у вас так, значит у вас ПРЛ» —
  недопустимы.
- Если статья вообще может коснуться темы травмы/суицида/зависимости —
  закладывать эту проверку заранее, а не постфактум.

## Тон и этика

- Не диагностировать читателя через текст статьи, только предлагать
  присмотреться к себе и рекомендовать очную консультацию.
- Не стыдить и не создавать чувство вины («вы сами виноваты, что не
  разобрались»). Наоборот — нормализовать («это не слабость», «это не ваша
  вина»).
- Не критиковать другие методы/коллег напрямую. Сравнение (например,
  схема-терапия vs КПТ) — только по фактам метода, не по компетентности людей.
- Мужской род от первого лица (автор — Григорий Котляревский). Проверять после
  генерации/редактирования — ИИ по умолчанию иногда сбивается на женский род.

## Форматирование контента (технические грабли)

- **Никаких сырых markdown-символов в HTML-контенте.** Контент хранится как
  HTML (`<p>`, `<strong>`, `<em>`, `<ul>` и т.д.), не markdown. Если текст
  собирается через LLM/скрипт из markdown-черновика — обязательно проверить,
  что **все** конструкции конвертированы (в частности когда-то проскочили
  необработанные `*текст*` вместо `<em>текст</em>` — на сайте это показалось бы
  буквальными звёздочками). Финальная проверка перед сохранением:
  `grep -n '\*[а-яА-Я]' на кусок контента` — не должно быть совпадений.
- Разрешённые теги в рендере — см. `ALLOWED_TAGS` в
  `webapp/src/pages/ArticlesPage.tsx` (DOMPurify). Если нужен новый тег
  (например, `blockquote`) — сначала добавить его туда и протестировать
  санитайзер, иначе контент просто вырежется молча.
- Диаграмма статьи — отдельное поле `diagramKey`, не часть текста. Не пытаться
  вписывать SVG прямо в `content` через редактор — WYSIWYG его выкинет.

## Структура и объём

- Каждая статья — с полноценным `description` (для карточки и превью), не
  дублирующим первый абзац дословно.
- `readMin` — честная оценка (примерно 200 слов/мин на русском), не занижать
  и не завышать для красоты.
- Один `slug` — латиницей, без изменений после публикации (ссылки уже могли
  разойтись/проиндексироваться).
- Обязательная последняя секция — мягкий призыв к действию (первая встреча /
  инструменты в приложении), без давления и без нескольких CTA подряд.

## Перед публикацией — чеклист

1. Клиническая точность: нет придуманных цифр/цитат, есть раздел «когда не
   подходит», где применимо.
2. Безопасность: если тема касается риска — есть актуальный кризисный контакт.
3. Тон: мужской род, не диагностирует читателя, не стыдит.
4. Формат: нет сырых markdown-символов в HTML, все использованные теги в
   белом списке DOMPurify.
5. `tsc --noEmit` и `vite build` (webapp) — чисто, если менялся код рендера.
