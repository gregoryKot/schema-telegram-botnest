## Инцидент 2026-07-20 — утечка в публичном репозитории

`scripts/railway_data.sql` — дамп боевой БД (17 юзеров, незашифрованные
клинические заметки, живые session-токены) — лежал в публичном репо с мая.
Файл удалён из рабочего дерева этим PR, но **остаётся в истории git**: для
полной зачистки нужна перезапись истории (`git filter-repo`) + force-push,
либо перевод репозитория в приватный. `.gitignore` теперь блокирует любые
дампы БД. Аудиты с ещё не закрытыми находками держатся локально (gitignore),
чтобы не публиковать карту уязвимостей.
